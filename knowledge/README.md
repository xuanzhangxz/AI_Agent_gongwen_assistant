# 知识库整理说明

## 三类资料分别放哪里

- [standards](/Users/zhangxuan/AI_Agent/knowledge/standards)：正式格式要求、行文规范、常见禁忌、单位内部标准。
- [exemplars](/Users/zhangxuan/AI_Agent/knowledge/exemplars)：过去写得好、被领导认可的成文案例。
- [templates](/Users/zhangxuan/AI_Agent/knowledge/templates)：固定文种结构或套表模板。

## 支持的文件格式

- `.md`
- `.txt`
- `.docx`

## 推荐文件头

如果方便，建议在文件开头增加下面这种头部，便于系统更准地召回：

```yaml
---
title: 全市重点工作进展汇报
doc_type: 工作进展汇报
audience: 局领导
tags: 月报, 重点项目, 风险问题
---
```

## 资料整理建议

1. 标准规范写成“原则 + 正例 + 反例”最有效。
2. 优秀案例尽量标注适用场景，比如周报、月报、专题汇报。
3. 同一类材料不要混放，避免好案例被误当成硬性规范。
4. 若某位领导有明显偏好用语，建议单独整理为一份标准文件。
