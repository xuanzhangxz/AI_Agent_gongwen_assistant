# 项目说明

## 一、项目名称

公文写作 Agent

## 二、建设背景

在政务办公场景中，向领导报送工作进展汇报、专题材料和阶段性总结是高频工作。此类材料通常具有较强的规范性要求，不仅需要符合公文写作格式标准，还需满足正式、严谨、审慎、逻辑清晰等基本文风要求。

在实际工作中，业务人员提交的初稿往往存在以下问题：

- 结构层次不清，重点不够突出。
- 行文风格不够正式，口语化表达较多。
- 对“进展、问题、下一步安排”的逻辑组织不稳定。
- 对内部格式标准、领导偏好和既有优秀案例复用不足。
- 需要反复审阅和修改，沟通成本较高。

基于以上痛点，本项目以“标准规范 + 优秀案例 + 大模型改写”为总体思路，建设一个可持续沉淀、可复用、可扩展的公文写作辅助工具。

## 三、建设目标

本项目的建设目标如下：

- 将分散的公文规范、模板和优秀案例沉淀为统一知识库。
- 建立标准化的粗稿改写流程，提升材料起稿质量。
- 降低人工反复修改成本，提高日常报文效率。
- 为后续建设图形化界面、自动化服务和办公集成能力提供基础底座。

## 四、适用范围

当前版本重点适用于以下材料类型：

- 工作进展汇报
- 阶段性工作总结
- 专题情况说明
- 领导审阅前的正式化润色稿

在补充模板和案例后，也可逐步扩展至请示、实施方案、报告、专报等文种。

## 五、总体方案

### 1. 知识沉淀

系统将知识分为三类：

- `standards`：公文规范、格式标准、常见禁忌、领导偏好要求。
- `templates`：固定文种结构模板。
- `exemplars`：已被认可的历史优秀案例。

### 2. 检索召回

系统根据输入草稿、文种、受众和目标信息，从知识库中检索最相关的片段，用于约束生成过程。

### 3. 生成复审

系统默认采用两阶段处理机制：

1. 首轮改写：生成符合规范和文风要求的正式稿。
2. 复审修订：围绕格式、逻辑、措辞和事实风险进行检查，并在必要时自动修订。

## 六、能力边界

为确保输出稳妥可控，当前版本遵循以下边界：

- 不编造数据、会议纪要、政策依据和领导指示。
- 不对缺失事实进行未经授权的补全。
- 不主动推断具有明确风险的信息。
- 对无法确认的内容优先使用审慎性表达。

因此，本项目定位为“公文写作辅助工具”，并不替代人工最终审核。

## 七、核心价值

本项目的主要价值体现在以下方面：

- 提升公文起稿质量，缩短从粗稿到可审稿的时间。
- 将单位写作经验制度化、资产化，减少对个人经验的依赖。
- 提高写作风格一致性，增强材料规范性和稳定性。
- 为后续扩展多文种、多部门、多接口协同提供基础。

## 八、当前交付内容

当前仓库已包含以下内容：

- 知识库目录结构及示例资料
- 知识加载与文档切片能力
- 中文轻量检索能力
- 公文改写与复审流程
- 命令行调用入口
- 基础测试用例

## 九、后续建设建议

建议下一阶段重点推进以下工作：

1. 补充真实业务规范与优秀案例，提升检索和生成质量。
2. 建设分文种模板库，形成多场景适配能力。
3. 建立评测样本集，对输出质量进行量化验证。
4. 对接图形化前端或办公机器人，实现更低门槛使用。
